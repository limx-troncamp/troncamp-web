赛制 · 四个「放开层级」· 顺序解锁

T1 → T2 → T3 → T4:逐档放开你能改的层级

同一个 hanging_mug 任务,四档对应四个「放开层级」:每爬一档,只多放开一类选手可以动手改的东西。 顺序解锁——前档达标才能交后档;主榜只排 T4 的纯成功率。

01 / 四档赛制

四个放开层级

放开层级逐档累进:T1 只处理数据 → T2 加数据增强 → T3 改网络结构 → T4 全放开冲主榜。顺序解锁,复用统一的 gate 逻辑(前档达标才解锁后档)。

档 名称 放开层级 选手能改 提交物 计分 评测环境
T1 入门通关 数据处理 处理主办方给的 raw bag + 跑通 3 路图像 π0.5 LoRA 仅权重 分级 SR ≥ 低阈值 → 达标满分 弱随机化
T2 数据增强 + 数据增强 图像级增强 + 可用专家脚本补采更多数据(见数据政策) 权重 + 代码 分级 SR ≥ 高阈值(须过 T1) 弱随机化
T3 网络结构 + 改网络结构 改 π0.5 网络结构(LoRA 位置 / rank、冻结策略、域不变层、action expert 容量;不换模型) 权重 + 代码 分级 SR ≥ 阈值(须过 T2) 弱随机化
T4 硬核主榜 全放开 前三档招式叠加,冲最高分 权重 + 代码 纯成功率,降序排名(主榜)(须过 T3) 全域 OOD

顺序解锁:没过 T1 不能交 T2,依此类推到 T4。每档各交各的模型、各自独立评测。T1 只交权重;T2 起须交权重 + policy 代码(评测机要用你的加载 / 动作头 / obs 处理代码才能把 ckpt 跑成动作,尤其 T3 改了结构)。

02 / 计分口径

T1–T3 达标满分,T4 主榜纯成功率

计分基于 hanging_mug 的 check_success;EVAL_REPEATS 生产取均值(默认 5)消 π0.5 采样噪声。

档指标判定 / 排名
T1分级成功率SR ≥ 低阈值 → 达标满分;未达标得 min(SR / 阈值, 1)
T2分级成功率SR ≥ 高阈值 → 达标满分;未达标 min(SR / 阈值, 1)(须先过 T1)
T3分级成功率SR ≥ 阈值 → 达标满分;未达标 min(SR / 阈值, 1)(须先过 T2)
T4成功率(主榜)须先过 T3 解锁;按纯成功率降序排名——分数越高名次越前
T1–T3 计分 = min(SR / 阈值, 1) （SR ≥ 阈值 即达标满分 1.0）
主榜(T4)= 成功率 降序 （纯成功率,无运动质量、无力质量分量)
分级成功率口径:逐 episode 取分级成功 {1 / 0.5 / 0} 后取均值(或按 hanging_mug check_success 落定二元口径,实现期定)。 运动质量不进选手计分——它只是主办方专家迭代时的内部验收标准。所有阈值由主办方赛前在 4090 上实测标定(题面先写「实测标定」)。
03 / 评测难度递进

弱随机化(T1–T3)vs 全域 OOD(T4)

评测难度按 RoboTwin 原生 task_config/demo_*.yml 旋钮逐档加码;EVAL_REPEATS 取均值(默认 5)。

T1 – T3 评测

弱随机化

  • 桌高随机
  • 轻度光照随机

介于 demo_clean 与 demo_randomized 之间的中间档——有扰动但不极端,用于把基础策略练到能稳定通关。

T4 评测 · 全卷重心

全域 OOD

  • 随机背景 random_background
  • 杂乱桌面 cluttered_table
  • 桌高 ±3cm 量级 random_table_height
  • 强光照随机 random_light
  • unseen 纹理 / 语言指令(RoboTwin 原生 seen / unseen 机制)

全开 + 没见过的纹理与指令,逼近真实分布外场景。

维度弱随机化(T1–T3)全域 OOD(T4)
桌高随机±3cm 量级随机
光照轻度随机强随机
背景固定随机背景
桌面干净杂乱(干扰物)
纹理 / 指令seenunseen(未见纹理 / 语言指令)

诊断量(不进题面计分、不上榜):鲁棒间隙 = SR弱随机 − SR全OOD,仅主办方私下用来标定 headroom。

04 / 数据政策

给定 bag 基线 + T2 起可补采增强

基线数据
主办方用专家(envs/hanging_mug.py::play_once)跑一遍采一份 raw bag 全卷共用下发。T1 = 处理这份给定 bag。
T2 数据增强
选手除图像级增强外,可用 starter/collect_expert.py 换随机化 / 换 seed 补采更多数据作为增强手段。
采集方式
采集走 SAVE-DURING-SEARCH 单趟(两阶段 replay 在 hanging_mug 上成功率约 0)。
seed 隔离
自采只在公开训练 seed 段进行,与评测 seed 表不相交;final 名次用私有 seed 表统跑,防 dev 过拟合。
