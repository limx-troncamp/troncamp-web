troncamp · hanging_mug 题面 赛制 T1–T4 流程 榜单 Starter ↗
参赛流程 · 自包含公开仓 · 自评 = 主办方评测同一套

从 clone 到上榜的干净流程

clone 一个公开自包含仓,一键装环境 → 用给定数据训 3 路图像 π0.5 → 本地自评 / 看 rollout 视频 → 按 track 提交。 主办方用同一仓 + 私有配置跑提交后端与赛末权威评测。

01 / 主线步骤

六步走完一档

clone 公开仓

git clone https://github.com/limx-troncamp/troncamp —— 自包含,平台评测子集(RoboTwin hanging_mug env + π0.5 训练推理 + client-server policy)已 vendored 锁版本。

一键装环境

setup/install.sh 装 conda(RoboTwin:curobo / sapien / mplib)+ uv(π0.5 / JAX);setup/fetch_assets.sh 拉 RoboTwin assets 与 π0.5 base ckpt(gs://);setup/env_check.py import 自检。

拿给定数据

主办方专家采的一份 raw bag 全卷共用。T1 直接处理它;T2 起可用 starter/collect_expert.py 换随机化 / 换 seed 补采增强(只在公开训练 seed 段)。

训练 3 路图像 π0.5

starter/train.py 把数据转成 3 相机 LeRobot 格式,跑基线 config 做 LoRA 微调(ema_decay=None,24G 可塞)。

本地自评 / 看 rollout

starter/eval_local.py(= eval/run_eval.py --seeds dev.json)在公开 dev 种子上本地打印分数;starter/watch_rollout.py 跑策略并渲一段近景 rollout 视频核查行为。

提交,自动评测上榜

submit/submit.py 带 token 按 --track 上传 ckpt(+ T2 起的 policy 代码);主办方 evald 在私有 final 种子上评测,匿名 token 榜出名次。

02 / 本地自评

本地分 = 主办方评测同一套 harness

评测逻辑公私共用一份:选手自评跑的就是主办方评测的同一个 run_eval.py,只把公开 dev 种子换成私有 final 种子。

# 本地在公开 dev 种子上自评出分(= run_eval.py --seeds dev.json) python starter/eval_local.py --ckpt ./ckpt.pt --track T1 # 跑策略 + 渲近景 rollout 视频(iter12 视角,看不全机器人但桌/杯/架/手清晰) python starter/watch_rollout.py --ckpt ./ckpt.pt --seed 0 --out rollout.mp4
评测 kit 数据流:起 π0.5 policy server(uv / JAX 环境,加载 ckpt)+ 起 RoboTwin sim client(conda:curobo / sapien),逐 seed 跑一局 hanging_mug,client ↔ server 走 openpi-client 协议传 obs(3 路图像)/ action;每局记分级 success,EVAL_REPEATS 次取均值聚合成 SR + 明细 JSON。跨环境 client-server 是为了解 JAX ↔ curobo 的依赖冲突。
03 / rollout 视频

近景视角核查策略行为

starter/watch_rollout.py 用 近景视角(iter12 风格)渲一段 rollout: 机器人不一定完整入画,但桌面、杯子、杯架、双手清晰可见,便于近距离核查抓取与挂杆对位的细节。

这与官网任务示意图用的全景视角(看得见整机器人 + 桌面)不同——示意图为说明语义,rollout 近景为调试行为。

04 / 提交

带 token 按 track 提交,匿名上榜

提交唯一通道 submit/submit.py,带 token(header)、按 --track 分题各交各的模型。无报名、无队名。

T1(入门通关):只交权重:

# T1:只交权重 python submit/submit.py --track T1 --token <你的令牌> --ckpt-file ./t1.pt

T2 / T3 / T4(开放):加 --code-dir,CLI 自动把 policy 代码打包上传:

# T2 / T3 / T4:权重 + policy 代码(自动打包,自动算 SHA256) python submit/submit.py --track T4 --token <你的令牌> \ --code-dir ./mycode --ckpt-file ./t4.pt
机制说明
token 鉴权提交带 token(已生成 500 个,22 字符 url-safe 全唯一)。后端校验 token ∈ 500 集 + per-token 每日限额(env DAILY_LIMIT)。无报名、无队名。
顺序解锁过 T1 才能交 T2,过 T2 才能交 T3,过 T3 才能交 T4。复用统一 gate 逻辑。
dev / final 双榜dev 公开(选手自评 / 可选 live dev 榜);final 私有(赛末权威排名,解锁前显示「赛末公布」)。
匿名榜每 token 取最佳分上榜,只显示 token 尾 6 位(如 …ROqRA)。榜上只有尾号 + 分数 + 时间,永不显示队名。
安全提交解包做 zip-slip / symlink 防护;ckpt 备案 SHA256。推理期只用白名单观测,读不到仿真内部状态 / 私有 seed。
05 / 最终交付物

开源代码 + 设计文档 + 演示视频

赛中按题用 submit.py 自动评测上榜;最终截止前另需提交以下,用于复现核验与评审。

开源代码 + 设计文档
完整代码(含采集脚本)开源到 GitHub,附技术设计文档——写明环境配置、运行指令、算法逻辑,须可复现。
演示视频
时长严格 ≤ 3 分钟,以屏幕录制 + 画中画(摄像头)形式:本人出镜讲解思路,并现场 Demo 运行效果。
